Performing a Merge

Since: M8
Last Modified: $Date: 2004/02/25 20:34:42 $

Scenario 1: Merge in Dialog

  1. Select a project and perform a Team>Merge.
  2. Choose the start and end tags in the wizard and finish.
  3. The merge will run in the background.
  4. Upon completion, the user is presented with a dialog of the incoming changes.
  5. Update the incoming changes and ensure they worked.

After updating, reperform the same merge. A no-changes dialog should be presented since the local contents match the end-poimt.

Scenario 1: Merge in Synchronize view

  1. Perform the previous scenario except for the last step
  2. Choose to remember the merge
  3. Ensure that the merge appears in the Synchronize View
  4. Exit and restart eclipse
  5. The merge should still exist in the Synchronize View
  6. Update to the incoming changes.
  7. Optionally release new changes to the merge endpoint and refresh the merge. The new updates should appear.
  8. Delete the merge from the synchronize view using the remove toolbar button
  9. The merge subscriber should be removed from the view.
