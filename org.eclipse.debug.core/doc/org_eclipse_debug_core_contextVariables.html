Context Variables

Identifier:
org.eclipse.debug.core.contextVariables

Since:
3.0

Description:
This extension point provides a mechanism for defining dynamic variables used in string substitution. The value of a context variable is resolved dynamically by a delegate (resolver), which can accept an argument.

Configuration Markup:

   <!ELEMENT extension (variable*)>



   <!ATTLIST extension


     point CDATA #REQUIRED


     id    CDATA #IMPLIED


     name  CDATA #IMPLIED


   >

  • point - a fully qualified identifier of the target extension point
  • id - an optional identifier of the extension instance
  • name - an optional name of the extension instance

   <!ELEMENT variable EMPTY>



   <!ATTLIST variable


     name        CDATA #REQUIRED


     resolver    CDATA #REQUIRED


     description CDATA #REQUIRED


   >

  • name - specifies a unique name for this variable.
  • resolver - specifies a Java class which implements org.eclipse.debug.core.stringsubstitution.IContextVariableResolver, which is used to determine the value of the variable
  • description - specifies a human-readable description of this variable
Examples:
The following is a definition of a context variable that resolves to the name of the selected resource: 
 
<extension point=
"org.eclipse.debug.core.contextVariables"
>

   
<variable 
      name=
"resource_name"

      expanderClass=
"com.example.ResourceNameExpander"

      description=
"The name of the selected resource"
>

   
</variable>

 
</extension>
